# 提交规范

## Commit Message 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

## Type 类型

| 类型 | 说明 |
|-----|------|
| `feat` | 新功能 |
| `fix` | 修复bug |
| `docs` | 文档更新 |
| `style` | 代码格式调整（不影响功能） |
| `refactor` | 代码重构 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具/依赖更新 |

## 示例

```
feat(auth): 添加用户登录功能

实现基于JWT的用户认证系统，包括：
- 登录接口
- Token刷新机制
- 登出功能

Closes #123
```

## 代码提交前检查

1. 确保代码可以正常编译/构建
2. 运行测试确保通过
3. 检查代码风格是否符合项目规范
4. 检查是否包含敏感信息（密码、密钥等）
5. 检查 `.gitignore` 是否正确配置
