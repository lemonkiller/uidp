# 产品设计文档：UIDP (User Interface Data Packet) 界面几何数据格式

## 1. 产品定位

**UIDP v5.3** 是一个极简的界面草图数据格式，用于在 AI 辅助开发流程中传递界面的几何信息。

> **核心定位**：纯可视化草图工具 → 输出形状位置数据 → AI 自行推断实现

### 简化原则

- ✅ 只保留形状几何数据（位置、大小、层级）
- ✅ 包含给 AI 的协议说明
- ✅ 包含基准分辨率信息（用于响应式换算）
- ✅ 包含画板信息（屏幕边界参考）
- ✅ **可选的组件语义标记**（帮助 AI 理解形状用途）
- ❌ 不包含风格指导
- ❌ 不包含布局系统

## 2. UIDP 文件格式 (v5.3)

### 2.1 完整示例

```uidp
# ═══════════════════════════════════════════════════════════
# UIDP PROTOCOL v5.3 - 界面几何数据格式
# ═══════════════════════════════════════════════════════════
#
# 【重要说明】
# - 画板(artboard)仅作为屏幕大小的基准参考，不应被实际绘制
# - 所有形状的坐标都是相对于画板左上角的偏移量（相对坐标）
# - 画板尺寸表示设计稿的原始分辨率，元素坐标在此坐标系内定义
# - 渲染时需要将画板内的元素按比例缩放到实际窗口大小
#
# 【坐标系统】
# - 所有坐标都是左上角原点 (0,0)
# - X轴向右递增，Y轴向下递增
# - 画板位置固定为 R:0,0,width,height（相对坐标系原点）
# - 元素坐标 = 元素在画板内的偏移量（已自动计算为相对坐标）
#
# META 字段说明：
#   canvas     - 实际画布尺寸 (从元素边界自动计算)
#   unit       - 单位类型 (px=像素)
#   preset     - 设备预设类型: mobile/tablet/desktop-hd/desktop-fhd/custom
#   presetSize - 预设参考尺寸 (用于计算缩放比例)
#
# 画板字段说明：
#   #0       - 画板序号固定为 0
#   T        - 类型: artboard=画板
#   R        - 矩形区域: x,y,width,height (像素值，x,y固定为0,0)
#   Z        - 层级: 固定为 0，画板始终在最底层
#   PRESET   - 预设类型: mobile/tablet/desktop-hd/desktop-fhd/custom
#
# 形状字段说明：
#   #N  - 序号标识符 (N=1,2,3...)
#   T   - 类型: rect=矩形 circle=圆形 line=线条 text=文本
#   R   - 矩形区域: x,y,width,height (像素值，相对于画板左上角)
#   Z   - 层级: 必须为纯数字（1,2,3...），数字越大越在上层
#   TXT - 文本内容 (仅 text 类型)
#   C   - 组件语义 (可选): button/input/select/checkbox/radio/textarea/switch/label/image/container
#
# 【渲染公式 - 等比缩放】
#   【强制】必须使用等比缩放，保持元素原始宽高比
#   scale = min(实际窗口宽度 / 画板宽度, 实际窗口高度 / 画板高度)
#   实际X = 元素相对X * scale
#   实际Y = 元素相对Y * scale
#   实际宽度 = 元素宽度 * scale
#   实际高度 = 元素高度 * scale
#
# AI 使用建议：
# - 根据画板 PRESET 确定目标设备类型
# - 【强制】初始屏幕窗口大小必须使用 presetSize 指定的基准分辨率
# - 使用上述渲染公式将相对坐标转换为实际像素位置
# - 根据形状类型选择合适的组件实现
# - 根据 C 字段（如有）确定组件语义
# ═══════════════════════════════════════════════════════════

META:canvas=2560x1440 | unit=px | preset=desktop-hd | presetSize=1280x720

#0 | T:artboard | R:0,0,1280,720 | Z:0 | PRESET:desktop-hd
#1 | T:rect | R:80,90,200,50 | Z:1 | C:input
#2 | T:rect | R:80,160,200,50 | Z:1 | C:button
#3 | T:text | R:60,40,300,40 | Z:2 | TXT:用户名 | C:label
#4 | T:line | R:0,240,1280,2 | Z:1
```

### 2.2 格式说明

#### PROTOCOL 区块

文件开头的注释部分，向 AI 说明如何解析本文件。

#### META 区块

包含画布信息：

```
META:canvas={width}x{height} | unit=px | preset={preset} | presetSize={width}x{height}
```

**字段说明**：

| 字段 | 说明 |
|------|------|
| canvas | 实际画布尺寸（从元素边界自动计算） |
| unit | 单位类型，固定为 px |
| preset | 设备预设类型: mobile/tablet/desktop-hd/desktop-fhd/custom |
| presetSize | 预设参考尺寸，用于计算缩放比例 |

#### ARTBOARD 区块（画板）

画板定义屏幕边界，作为布局参考：

```
#0 | T:artboard | R:0,0,{width},{height} | Z:0 | PRESET:{preset}
```

**字段说明**：

| 字段 | 必需 | 说明 |
|------|------|------|
| #0 | 是 | 画板序号固定为 0 |
| T | 是 | 固定为 `artboard` |
| R | 是 | 画板尺寸 width,height（像素），位置固定为 0,0 |
| Z | 是 | 固定为 0，画板始终在最底层 |
| PRESET | 是 | 预设类型，决定画板尺寸 |

**预设类型与尺寸**：

| 预设 | 尺寸 | 比例 | 适用场景 |
|------|------|------|----------|
| mobile | 375x667 | 9:16 | iPhone SE/8 尺寸 |
| tablet | 768x1024 | 3:4 | iPad 竖屏 |
| desktop-hd | 1280x720 | 16:9 | 常见桌面显示器（HD） |
| desktop-fhd | 1920x1080 | 16:9 | 全高清桌面显示器（FHD） |
| custom | 用户自定义 | - | 特殊需求 |

#### SHAPES 区块

每行一个形状，格式：

```
#{n} | T:{type} | R:{x},{y},{w},{h} | Z:{z-index} [| TXT:{text}] [| C:{component}]
```

**字段说明**：

| 字段 | 必需 | 说明 |
|------|------|------|
| #N | 是 | 序号标识符 (N=1,2,3...)，用于标识和引用形状 |
| T | 是 | 形状类型：rect/circle/line/text |
| R | 是 | 矩形区域 x,y,width,height（像素，相对于画板左上角） |
| Z | 是 | 层级，**强制要求纯数字**（如 1, 2, 3...），数字越大越在上层 |
| TXT | 否 | 文本内容（仅 text 类型） |
| C | 否 | 组件语义标记（可选），见下表 |

**形状类型**：

| 类型 | 说明 |
|------|------|
| rect | 矩形 |
| circle | 圆形/椭圆 |
| line | 线条 |
| text | 文本 |

**组件语义类型（C 字段）**：

| 组件类型 | 说明 | 适用形状 |
|----------|------|----------|
| button | 按钮 | rect, circle |
| input | 输入框 | rect |
| select | 下拉选择 | rect |
| checkbox | 复选框 | rect, circle |
| radio | 单选框 | circle |
| textarea | 多行文本输入 | rect |
| switch | 开关 | rect, circle |
| label | 标签/说明文字 | text |
| image | 图片占位 | rect |
| container | 容器/卡片 | rect |

## 3. 技术架构

### 3.1 技术栈选择

| 技术 | 用途 | 选择理由 |
|------|------|----------|
| **Tauri v2** | 桌面应用框架 | 包体积小(~5MB)、性能好、Windows 原生支持 |
| **React 19** | UI 框架 | 生态成熟、Excalidraw 集成方便 |
| **TypeScript** | 类型系统 | 开发体验好、减少运行时错误 |
| **@excalidraw/excalidraw** | 核心画布 | 功能完善、手绘风格、开源免费 |

**为什么不用 Electron？**
- Electron 包体积太大（100MB+）
- Tauri 使用系统 WebView，体积仅 ~5-10MB
- Tauri 内存占用更低、启动更快

### 3.3 数据转换映射

| Excalidraw 类型 | UIDP 类型 | 说明 |
|----------------|-----------|------|
| rectangle | rect | 矩形 |
| ellipse | circle | 圆形/椭圆 |
| line | line | 线条 |
| text | text | 文本 |
| diamond | rect | 菱形映射为矩形 |
| arrow | line | 箭头映射为线条 |
| frame | artboard | 画板（面积最大的 Frame 被识别为画板） |

**画板识别逻辑**：
- Excalidraw 中创建 Frame（快捷键 F）
- 导出时选择面积最大的 Frame 作为画板
- 转换为 `T:artboard` | PRESET:{preset}
- 画板位置固定为 R:0,0,width,height（相对坐标系原点）

**组件语义标记逻辑**：
- 用户选中形状后，在顶部工具栏选择组件类型
- 通过 `customData: { uidpComponent: "button" }` 存储语义
- 导出时读取并转换为 `C:button` 字段
- 不选择时则不输出 C 字段，保持向后兼容

**坐标系统**：
- 所有形状的坐标都是相对于画板左上角的偏移量（相对坐标）
- 椭圆和菱形的中心点坐标会自动转换为左上角坐标
- 渲染时需要使用等比缩放公式将相对坐标转换为实际像素位置

## 4. 工作流程

### 4.1 基础工作流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ ① 选择预设  │ ──→ │ ② 绘制草图  │ ──→ │ ③ 导出.uidp │ ──→ │ ④ AI 理解   │
│ (mobile等)  │     │(Frame工具+F)│     │  (几何数据)  │     │  自行实现    │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

1. **选择预设**：在顶部工具栏选择设备预设（移动端/平板/桌面）或输入自定义尺寸，决定画板尺寸
2. **绘制草图**：使用 Frame 工具（F键）创建画板，在画板内绘制界面元素
3. **导出数据**：导出为 `.uidp` 文件（包含画板信息和几何数据）
4. **AI 实现**：AI 读取文件，根据画板预设和形状位置自行推断布局，在目标平台实现

### 4.2 带组件语义的工作流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ ① 选择预设  │ ──→ │ ② 绘制草图  │ ──→ │ ③ 标记组件  │ ──→ │ ④ 导出.uidp │
│ (mobile等)  │     │(Frame工具+F)│     │(工具栏选择) │     │(含C字段)    │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
                                                                  │
                                                                  ▼
┌─────────────┐     ┌─────────────┐
│ ⑥ 代码实现  │ ←── │ ⑤ AI 理解   │
│(目标平台)   │     │(几何+语义)  │
└─────────────┘     └─────────────┘
```

**组件标记详细步骤**：

1. **绘制形状**：使用矩形工具画一个长方形（作为输入框）
2. **选中形状**：点击选中该矩形
3. **选择组件类型**：在顶部工具栏的「组件类型」下拉框选择 "input"（输入框）
4. **继续标记**：重复步骤 1-3，标记其他组件（按钮、标签等）
5. **导出**：点击导出，生成的 UIDP 文件包含 `C:input` 等字段

### 5.2 组件属性面板

**位置**：集成在顶部工具栏中

**内容**：
- **当前选中**：显示选中形状的类型（矩形/圆形/文本等）
- **组件类型选择器**：下拉框，选项包括：
  - 无（默认，不输出 C 字段）
  - button（按钮）
  - input（输入框）
  - select（下拉选择）
  - checkbox（复选框）
  - radio（单选框）
  - textarea（多行文本）
  - switch（开关）
  - label（标签）
  - image（图片）
  - container（容器）
- **清除标记按钮**：移除当前形状的组件语义

**交互逻辑**：
- 未选中形状时，显示「请选择一个形状」
- 选中形状后，读取其 `customData.uidpComponent` 显示当前值
- 用户选择新值后，通过 `excalidrawAPI.updateScene()` 更新
- 画板（Frame）自动识别，无需标记组件类型

1. **极简**：格式简单，AI 易于理解
2. **快速**：基于 Excalidraw，开发周期短
3. **轻量**：Windows 安装包仅 ~5-10MB
4. **灵活**：AI 有充分的自由度来决定如何实现
5. **专注**：人类专注布局，AI 专注实现

7. **响应式支持**：相对坐标系统，支持等比缩放适配不同屏幕
8. **多语言**：支持中文/英文界面切换

## 8. 未来规划

### 8.1 UIDP 文件导入功能（已实现）

#### 功能概述

UIDP Editor 现已支持 `.uidp` 文件的导入功能，实现了完整的工作流程闭环：

```
绘制草图 → 导出 .uidp → AI 理解实现 → 导入修改 → 继续迭代
```

#### 功能价值

| 场景 | 价值 |
|------|------|
| **AI 协作闭环** | 导出 → AI 修改 `.uidp` → 导入查看效果 → 继续迭代 |
| **批量修改** | 用文本编辑器或脚本批量改文案、调间距 |
| **版本管理** | `.uidp` 文件可纳入 git 管理，支持回滚和对比 |
| **设计模板** | 保存常用布局为 `.uidp` 模板，快速导入复用 |

#### 工作流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  绘制草图    │ ──→ │  导出.uidp   │ ──→ │  给AI修改    │
│(UIDP Editor)│     │             │     │             │
└─────────────┘     └─────────────┘     └──────┬──────┘
       ↑                                         │
       └─────────────────────────────────────────┘
                    导入修改后的文件
```

**AI 可修改的内容**：
- 调整元素位置和大小（`R:x,y,w,h`）
- 修改文本内容（`TXT:xxx`）
- 更改组件语义（`C:button/input/...`）
- 调整层级（`Z:N`）

#### 技术实现

**解析策略**：
1. 跳过注释行（`#` 开头但不是 `#数字` 的行）
2. 解析 `META` 行获取画布信息
3. 逐行解析元素数据（`#N` 格式）
4. 转换为 Excalidraw 元素格式

**类型映射**：

| UIDP 类型 | Excalidraw 类型 | 说明 |
|-----------|-----------------|------|
| `rect` | `rectangle` | 矩形 |
| `circle` | `ellipse` | 圆形/椭圆 |
| `line` | `line` | 线条 |
| `text` | `text` | 文本 |
| `artboard` | `frame` | 画板 |

**坐标处理**：
- 导出：绝对坐标 → 相对坐标（相对于画板）
- 导入：相对坐标 → 绝对坐标（还原到画布位置）

**实现文件**：
- `src/utils/uidp.ts` - `parseUIDP()` 和 `convertFromUIDP()` 函数
- `src/App.tsx` - `handleImportFromFile()` 函数

#### 交互设计

顶部工具栏「导入 .uidp 文件」按钮：

```
┌─────────────────────────────────────────────────────────────────────┐
│  基准分辨率: [移动端 ▼] [自定义尺寸] [导出] [📋复制] [📂导入] [?] [🌐] │
└─────────────────────────────────────────────────────────────────────┘
```

**导入行为**：
- 清空当前画布，加载导入的内容
- 自动滚动到导入内容的位置
- 根据 META 信息恢复设备预设选择

#### 边界情况处理

| 情况 | 处理方案 |
|------|----------|
| 文件格式错误 | 显示友好的错误提示「文件格式无效」|
| 缺少画板 | 自动创建默认画板 |
| 版本不兼容 | 基于 v5.3 协议版本解析 |
| 元素 ID 冲突 | 重新生成唯一 ID |
