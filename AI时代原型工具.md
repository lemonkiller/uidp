# 产品设计文档：UIDP (User Interface Data Packet) 界面几何数据格式

## 1. 产品定位

**UIDP v5.0** 是一个极简的界面草图数据格式，用于在 AI 辅助开发流程中传递界面的几何信息。

> **核心定位**：纯可视化草图工具 → 输出形状位置数据 → AI 自行推断实现

### 简化原则

- ✅ 只保留形状几何数据（位置、大小、层级）
- ✅ 包含给 AI 的协议说明
- ❌ 不包含风格指导
- ❌ 不包含布局系统
- ❌ 不包含组件语义

---

## 2. UIDP 文件格式 (v5.0)

### 2.1 完整示例

```uidp
# ═══════════════════════════════════════════════════════════
# UIDP PROTOCOL v5.0 - 界面几何数据格式
# ═══════════════════════════════════════════════════════════
#
# 本文件包含界面草图的几何数据，每行一个形状。
#
# 字段说明：
#   ID  - 唯一标识符
#   T   - 类型: rect=矩形 circle=圆形 line=线条 text=文本
#   R   - 矩形区域: x,y,width,height (像素值)
#   Z   - 层级: 数字越大越在上层
#   TXT - 文本内容 (仅 text 类型)
#
# AI 使用建议：
# - 根据形状位置和层级推断界面结构
# - 根据形状类型选择合适的组件实现
# - 根据相对位置推断布局关系
# ═══════════════════════════════════════════════════════════

ID:s1 | T:rect | R:100,200,120,40 | Z:1
ID:s2 | T:circle | R:300,220,20,20 | Z:1
ID:s3 | T:text | R:80,100,200,30 | Z:2 | TXT:Title
ID:s4 | T:line | R:0,150,400,2 | Z:0
```

### 2.2 格式说明

#### PROTOCOL 区块

文件开头的注释部分，向 AI 说明如何解析本文件。

#### SHAPES 区块

每行一个形状，格式：

```
ID:{id} | T:{type} | R:{x},{y},{w},{h} | Z:{z-index} [| TXT:{text}]
```

**字段说明**：

| 字段 | 必需 | 说明 |
|------|------|------|
| ID | 是 | 唯一标识符 |
| T | 是 | 形状类型：rect/circle/line/text |
| R | 是 | 矩形区域 x,y,width,height（像素） |
| Z | 是 | 层级，越大越在上层 |
| TXT | 否 | 文本内容（仅 text 类型） |

**形状类型**：

| 类型 | 说明 |
|------|------|
| rect | 矩形 |
| circle | 圆形/椭圆 |
| line | 线条 |
| text | 文本 |

## 3. 技术架构

### 3.1 技术栈选择

| 技术 | 用途 | 选择理由 |
|------|------|----------|
| **Tauri** | 桌面应用框架 | 包体积小(~5MB)、性能好、Windows 原生支持 |
| **React 18** | UI 框架 | 生态成熟、Excalidraw 集成方便 |
| **TypeScript** | 类型系统 | 开发体验好、减少运行时错误 |
| **@excalidraw/excalidraw** | 核心画布 | 功能完善、手绘风格、开源免费 |

**为什么不用 Electron？**
- Electron 包体积太大（100MB+）
- Tauri 使用系统 WebView，体积仅 ~5-10MB
- Tauri 内存占用更低、启动更快

### 3.2 整体架构

```
┌─────────────────────────────────────────┐
│        Windows 桌面应用 (.exe/.msi)      │
├─────────────────────────────────────────┤
│                                         │
│   ┌─────────────────────────────────┐  │
│   │      @excalidraw/excalidraw      │  │
│   │           (核心画布)              │  │
│   │   - 矩形、圆形、线条、文本         │  │
│   │   - 选择、拖拽、缩放              │  │
│   │   - 层级、对齐                    │  │
│   │   - 撤销重做                      │  │
│   └─────────────────────────────────┘  │
│                                         │
│   ┌─────────────────────────────────┐  │
│   │         导出面板                 │  │
│   │    - 导出 .uidp                 │  │
│   │    - 复制到剪贴板                │  │
│   └─────────────────────────────────┘  │
│                                         │
├─────────────────────────────────────────┤
│   Tauri (Rust) - 仅处理文件对话框和读写   │
│   - 打开/保存文件对话框                  │
│   - 读取/写入 .uidp 文件                │
└─────────────────────────────────────────┘
```

### 3.3 数据转换映射

| Excalidraw 类型 | UIDP 类型 | 说明 |
|----------------|-----------|------|
| rectangle | rect | 矩形 |
| ellipse | circle | 圆形/椭圆 |
| line | line | 线条 |
| text | text | 文本 |
| diamond | rect | 菱形映射为矩形 |
| arrow | line | 箭头映射为线条 |

## 4. 工作流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  ① 绘制草图  │ ──→ │ ② 导出.uidp │ ──→ │ ③ AI 理解   │
│  (Excalidraw)│     │  (几何数据)  │     │  自行实现    │
└─────────────┘     └─────────────┘     └─────────────┘
```

1. **绘制草图**：在 Excalidraw 画布上绘制界面草图
2. **导出数据**：导出为 `.uidp` 文件（仅包含几何数据）
3. **AI 实现**：AI 读取文件，根据形状自行推断用途和样式，在目标平台实现

## 6. 优势

1. **极简**：格式简单，AI 易于理解
2. **快速**：基于 Excalidraw，开发周期短（4天）
3. **轻量**：Windows 安装包仅 ~5-10MB
4. **灵活**：AI 有充分的自由度来决定如何实现
5. **专注**：人类专注布局，AI 专注实现
